<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>

            <executions>
                <execution>
                    <phase>process-sources</phase>
                    <goals>
                        <goal>run</goal>
                    </goals>
                    <configuration>
                        <target>
                            <!-- Define the directory and include Java files -->
                            <fileset dir="${project.basedir}/src/main/java" includes="**/*.java" id="java-files"/>

Step#3: Extract the Certificate and Private Key from the output file (response.txt).

Note: Stream the output to a text file since it will take time for vault to generate a new certificate and respond. CURL command will add some download logs that will mess up the output when you copy from console.

Step#4: Replace "\n" characters in the extracted certificate and private keys with newline characters. Otherwise, the openssl and keytool will not recognize them as valid certificate files. I have provided the below instructions earlier.

Steps to generate Keystore and Truststore from Certificate

Request for openssl to be installed from Software Catalog. Search for "OpenSSL" and request for 64-bit windows version or appropriate one for Mac.
Download the USBank Root CA, certificate and the private key

Consider USBankRootCA.cer as the downloaded USBank Root CA certificate file
Consider reference-data-service-dev-tls.cer as the downloaded Certificate file
Consider reference-data-service-dev-tls_PK as the downloaded Private Key file
Using openssl, convert USBank Root CA to correct format:

openssl x509 -inform PEM -in .\USBankRootCA.cer -out .\USBankRootCA.crt

Using openssl, convert Certificate to correct format:

openssl x509 -inform PEM -in .\reference-data-service-dev-tls.cer -out .\reference-data-service-dev-tls.crt

Using openssl, create a PKCS12 Keystore using the Key

openssl pkcs12 -export -inkey reference-data-service-dev-tls_PK -in reference-data-service-dev-tls.crt -out reference-data-service-dev.us.bank-dns.com.pfx -name reference-data-service-dev.us.bank-dns.com.pfx

Using keytool, create a JKS keystore. Note the password you use when prompted.

keytool -importkeystore -srckeystore reference-data-service-dev.us.bank-dns.com.pfx -srcstoretype pkcs12 -destkeystore reference-data-service-dev-keystore.jks -deststoretype jks

Using keytool, Create a truststore that includes USBank Root CA root certificate. Note the password you use when prompted.

keytool -import -file USBankRootCA.crt -keystore reference-data-service-dev-truststore.jks -storetype jks -alias UsRootCA

Use the above generated JKS keystore and truststore to connect to Kafka.
